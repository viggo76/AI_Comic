很好，现在进入 **Production Bible（制作圣经）** 的核心部分。

**Shot Bible（镜头语言圣经）** 的作用：

> 规定整部 AI 漫剧“怎么拍”。

它解决的问题：

- 为什么同样的角色，AI生成出来像不同作品？
- 为什么有些镜头像小说插图，没有影视感？
- 为什么爽点不够强？

因为 AI 漫剧不是“生成图片”，而是：

> 用镜头语言控制观众情绪。

下面建立《重生后，我绑定了万亿神豪系统》的 **Shot Bible V1.0**。

------

# Shot Bible

## 《重生后，我绑定了万亿神豪系统》

Version 1.0

------

# 1. Overall Visual Philosophy（整体视觉理念）

## 核心关键词

```
AAA game cinematic
+
Chinese urban drama
+
Korean webtoon storytelling
+
Hollywood cinematic camera language
```

------

## 视觉目标

不是漫画感。

而是：

> “正在观看一部3D游戏CG短剧。”

------

## 镜头原则

每一个镜头必须服务于：

1. 信息传递
2. 情绪增强
3. 爽点释放
4. 悬念制造

禁止：

- 单纯人物站立说话
- 长时间静态画面
- 无意义环境展示

------

# 2. Camera Language（摄影语言）

------

# A. 景别系统 Shot Size

------

# ① Extreme Close Up（ECU）

## 超特写

### 用途：

最高情绪点。

适合：

- 震惊
- 愤怒
- 系统觉醒
- 身份曝光

------

### 画面：

只显示：

- 眼睛
- 嘴角
- 手部动作

------

### Prompt:

```
extreme close-up shot,
detailed facial expression,
realistic skin texture,
cinematic lighting,
shallow depth of field,
85mm lens
```

------

### 使用场景：

林川：

系统开启。

顾天宇：

发现被反杀。

沈清禾：

第一次认可林川。

------

# ② Close Up（CU）

## 特写

### 用途：

人物心理。

------

展示：

头部+肩膀。

------

Prompt:

```
close-up portrait shot,
emotional expression,
cinematic lighting,
soft background blur
```

------

# ③ Medium Close Up（MCU）

## 中近景

### 用途：

对话。

商业谈判。

------

范围：

胸口以上。

------

Prompt:

```
medium close-up shot,
character dialogue,
natural body language,
cinematic composition
```

------

# ④ Medium Shot（MS）

## 中景

### 用途：

人物动作。

------

范围：

腰部以上。

------

适合：

- 走路
- 打电话
- 争论

------

# ⑤ Full Shot（FS）

## 全身

### 用途：

展示身份变化。

例如：

普通林川：

出租屋。

↓

高级西装林川：

董事会。

------

Prompt:

```
full body cinematic shot,
hero composition,
dramatic lighting
```

------

# ⑥ Wide Shot（WS）

## 广角远景

### 用途：

建立世界。

------

例如：

- 城市
- 公司
- 酒会

------

Prompt:

```
wide cinematic establishing shot,
modern city background,
dramatic atmosphere
```

------

# 3. Camera Angle（机位）

------

# ① Eye Level（平视）

用途：

正常交流。

占：

50%

------

Prompt:

```
eye level camera angle,
natural perspective
```

------

# ② Low Angle（低机位）

用途：

制造权力感。

------

使用：

林川逆袭。

顾天宇登场。

董事会。

------

Prompt:

```
low angle hero shot,
powerful presence,
cinematic perspective
```

------

# ③ High Angle（高机位）

用途：

弱势。

失败。

孤独。

------

例如：

EP01死亡。

EP19失败。

------

Prompt:

```
high angle shot,
small figure,
dramatic loneliness
```

------

# ④ Over Shoulder（肩后镜头）

用途：

对抗。

------

例如：

林川 vs 顾天宇。

------

Prompt:

```
over the shoulder shot,
two character confrontation,
cinematic depth
```

------

# 4. Movement Language（镜头运动）

------

# ① Push In（推进）

⭐⭐⭐⭐⭐

最重要。

用途：

情绪升级。

------

例：

系统出现：

镜头慢慢推进林川眼睛。

------

Prompt:

```
slow camera push in,
dramatic reveal,
cinematic movement
```

------

# ② Pull Out（拉远）

用途：

展示孤独。

------

例如：

林川站在城市中。

------

# ③ Tracking Shot（跟拍）

用途：

人物登场。

------

例如：

顾天宇下车。

------

Prompt:

```
cinematic tracking shot,
walking character,
dynamic camera movement
```

------

# ④ Orbit Shot（环绕）

用途：

高级感。

------

例如：

林川第一次进入豪宅。

------

Prompt:

```
360 degree orbit camera,
hero reveal,
cinematic
```

------

# ⑤ Slow Motion（慢动作）

用途：

爽点。

------

例如：

银行卡落桌。

车钥匙放下。

------

Prompt:

```
cinematic slow motion,
dramatic moment,
high detail
```

↓

UI特写

↓

### ④ 反击

Low Angle

↓

### ⑤ 对方震惊

ECU

# 7. System Shot Rules（系统镜头）

系统出现不能像弹窗。

应该像：

科幻HUD。

------

固定：

镜头：

黑屏↓

数据流↓

金色文字↓

林川眼睛反射。

------

Prompt:

```
futuristic golden holographic system interface,
reflected in character eyes,
sci-fi HUD,
cinematic lighting
```

------

# 8. Episode Rhythm Rule（单集镜头比例）

90秒一集：

建议：

| 镜头类型     | 比例 |
| ------------ | ---- |
| 人物情绪特写 | 30%  |
| 剧情动作     | 30%  |
| 环境建立     | 20%  |
| 系统UI       | 10%  |
| 爽点慢动作   | 10%  |

------

# 9. 爽剧镜头公式

每一集：

固定：

```
0-5秒
强冲突镜头

5-30秒
压迫镜头

30-60秒
反击镜头

60-80秒
爽点镜头

80-90秒
悬念镜头
```
